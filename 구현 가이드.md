### 백엔드
- **Supabase** (프로젝트 Ref: `egwmkpplnzypkbedasrs`)
  - PostgreSQL DB — orders, kakao_tokens 테이블
  - RPC 함수 — create_order (anon), create_license (service_role)
  - Database Trigger — orders INSERT → Edge Function 호출
  - Edge Functions (Deno/TypeScript) — 알림 3중 발송
  - pg_net 확장 — HTTP 호출
  - supabase_vault — service_role_key 안전 보관

- **GitHub Pages** + 커스텀 도메인 `blog.pluscoach.co.kr`

### Phase 3: DB 스키마 ✅ (2026-04-09)

**목적:** orders 테이블, kakao_tokens 테이블, RPC 함수들, Database Trigger 생성.

**Migration 5개 작성 및 push 완료:**

#### 001: orders 테이블
```sql
-- public.orders
-- id, name, email, plan, amount, status, ip, user_agent, order_code, license_key, created_at, updated_at
-- plan CHECK: 'free_trial', 'monthly', 'full_package'
-- status 기본값: '결제대기'
-- RLS 활성화 (정책 없음 = anon 직접 접근 불가)
-- updated_at 자동 갱신 트리거 (handle_updated_at)
-- 인덱스: idx_orders_email_plan
```

#### 002: create_order RPC
```sql
-- create_order(p_name, p_email, p_plan, p_amount, p_ip, p_user_agent) → JSON
-- 무료체험 이메일 중복 체크 (RAISE EXCEPTION 한글 메시지)
-- order_code 자동 생성: 이름-랜덤4자리 (예: 홍길동-A3X7)
-- 무료체험: status '처리중' / 유료: status '결제대기'
-- GRANT: anon, authenticated
```

#### 003: create_license RPC
```sql
-- create_license(p_buyer_name, p_plan, p_order_code) → TEXT (라이선스 키)
-- 플랜별 만료: free_trial 24시간, monthly 30일, full_package 365일
-- 키 형식: BAF-홍길동A3X7-X9K2P8
-- licenses 테이블에 INSERT (기존 테이블 구조 변경 0)
-- GRANT 없음 → service_role(Edge Function)만 호출 가능
```

#### 004: kakao_tokens 테이블
```sql
-- public.kakao_tokens
-- id (단일 행 제약: CHECK id = 1), access_token, refresh_token, access_token_expires_at, updated_at
-- RLS 활성화 (service_role만 접근)
```

#### 005: Database Trigger + Vault
```sql
-- pg_net 확장 활성화
-- supabase_vault 확장 활성화
-- notify_new_order() 트리거 함수:
--   vault.decrypted_secrets에서 service_role_key 조회
--   net.http_post로 Edge Function 호출 (Authorization: Bearer 포함)
--   order 데이터를 {record: {...}} 형태로 전달
-- on_order_inserted 트리거: orders INSERT 시 실행
```

**Vault 설정:** `vault.create_secret('실제키', 'service_role_key')` 완료 (UUID: `261781a3-...`)

**Migration 파일명 이슈:** 처음 `20260409_001_` 형식으로 생성했으나, Supabase가 버전 번호를 숫자 prefix로 추출해서 5개가 모두 `20260409`로 충돌. `20260409000100`, `20260409000200` 등 고유 타임스탬프로 변경하여 해결.

**검증 결과:**
- 테이블 3개 존재 확인 (orders, kakao_tokens, licenses 원본 유지)
- RPC 5개 (기존 3 + 신규 2) 확인
- 트리거 2개 확인
- create_order 실제 호출 JSON 정상 반환
- 중복 차단 정상 작동

### Phase 5-A: Edge Function 코드 작성 ✅ (2026-04-09, 배포 미진행)

**목적:** 주문 INSERT 시 카카오톡 + 텔레그램 + Resend 이메일 3개 동시 발송하는 Edge Function.

**작성 파일 4개:**

#### `on-new-order/index.ts` (73줄)
- Database Trigger가 보내는 `{record: {...}}` 페이로드 수신
- `Promise.allSettled`로 3개 알림 병렬 발송
- 각 발송 결과 console.log (디버깅용)
- 무료체험 분기: 주석으로 표시 (Phase 6에서 구현)
- 에러여도 항상 200 반환 (Trigger 재시도 방지)

#### `_shared/kakao.ts` (127줄)
- `getValidAccessToken()` — kakao_tokens 테이블에서 토큰 읽기, 만료 5분 전이면 자동 갱신
- `refreshAccessToken()` — refresh_token으로 새 access_token 발급 + DB 저장, refresh_token 갱신 시 함께 저장
- `sendKakaoNotification()` — "나에게 보내기" API 호출
- Supabase client: `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY`로 초기화
- 카톡 알림 link URL: `https://blog.pluscoach.co.kr`

#### `_shared/telegram.ts` (41줄)
- `sendTelegramNotification()` — `api.telegram.org/bot{TOKEN}/sendMessage` 호출
- 환경변수: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`

#### `_shared/resend.ts` (75줄)
- `sendOrderConfirmationEmail()` — `api.resend.com/emails` 호출
- HTML 이메일 템플릿: 주문 정보 + 플랜별 분기 (무료체험: 결제 안내 생략, 유료: 결제 안내 포함)
- 발신: `BlogAutoFriends <noreply@blog.pluscoach.co.kr>`
- 환경변수: `RESEND_API_KEY`, `RESEND_FROM_EMAIL`

**배포 보류 사유:** 사전 준비물 미완 (카카오 앱, 텔레그램 봇, Resend 계정)

## 4. DB 스키마 (운영 현황)

### 테이블

| 테이블 | 상태 | 용도 |
|---|---|---|
| `licenses` | 기존 (절대 건드리지 않음) | 프로그램 라이선스 관리, 9개 컬럼 |
| `orders` | Phase 3에서 생성 | 주문/신청 관리 |
| `kakao_tokens` | Phase 3에서 생성 | 카카오 토큰 관리 (단일 행) |

### RPC 함수

| 함수 | 권한 | 용도 |
|---|---|---|
| `verify_and_activate(key, machine_id)` | anon | 프로그램 라이선스 검증 (기존) |
| `update_session(key, machine_id)` | anon | 프로그램 세션 갱신 (기존) |
| `clear_session(key)` | anon | 프로그램 세션 정리 (기존) |
| `create_order(p_name, p_email, p_plan, p_amount, p_ip, p_user_agent)` | anon | 랜딩페이지 주문 (신규) |
| `create_license(p_buyer_name, p_plan, p_order_code)` | service_role | 라이선스 발급 (신규) |

### Trigger

| 트리거 | 테이블 | 이벤트 | 동작 |
|---|---|---|---|
| `on_orders_updated` | orders | BEFORE UPDATE | updated_at 자동 갱신 |
| `on_order_inserted` | orders | AFTER INSERT | Edge Function 호출 (net.http_post) |

## 8. 보안 설계

| 항목 | 구현 |
|---|---|
| anon key 노출 | OK (프론트엔드 공개, RPC만 호출 가능) |
| service_role key | Edge Function Secrets + Supabase Vault에만 저장 |
| orders 테이블 직접 접근 | RLS 차단 (정책 없음), create_order RPC만 허용 |
| kakao_tokens 테이블 | RLS 차단, service_role만 접근 |
| licenses 테이블 | 기존 RLS 그대로 유지 |
| create_license 호출 | anon GRANT 없음, service_role만 가능 |
| DB Trigger → Edge Function | Authorization: Bearer + Vault에서 읽은 service_role_key |
| 무료체험 악용 방지 | 이메일 중복 체크 (create_order RPC 내부) |
