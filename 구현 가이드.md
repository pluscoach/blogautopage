## 1. 기술 스택 & 구조

### 기본 구조
- **단일 HTML 파일** (`index.html`) — CSS/JS 인라인

### Tailwind 커스텀 설정
```js
tailwind.config = {
    theme: {
        extend: {
            fontFamily: { sans: ['Pretendard', 'sans-serif'] },
            colors: {
                naver: '#03C75A',
                naverHover: '#02b350',
                naverLight: '#E8F5E9',
                dark: '#1A1A1A',
            },
            animation: {
                'pulse-slow': 'pulse 3s cubic-bezier(0.4, 0, 0.6, 1) infinite',
            }
        }
    }
}
```

## 3. 섹션별 구현 상세

---

### SECTION 1: 히어로

**디자인**: 태그라인 + 임팩트 헤드라인 + 수치 뱃지 + AI 데모 UI

**좌측 텍스트:**
- 태그라인 뱃지: "네이버 블로그 서이추 자동화" (`bg-green-50 border-green-200 rounded-full`)
- 헤드라인: "AI가 하루 100명과 소통하는 동안, 당신은 글만 쓰세요." (`text-4xl~[3.4rem] font-extrabold`)
- 수치 뱃지 3개: 하루 100명 / 수락률 87% / 990원 (pill 형태)
- CTA: "0원으로 24시간 무료 체험하기"
- 스크롤 힌트: "↓ 어떻게 작동하는지 확인하기"

**우측 AI 데모:**
- 맥 스타일 윈도우 (빨/노/초 dots)
- 포스팅 분석 → 태그 추출 → AI 댓글 타이핑 애니메이션 (3개 댓글 순환)
- 소통 진행률 카운터 (0→100)
- 전체 float 애니메이션

**애니메이션:** `blink`, `fadeInUp` (hero-fade-d1~d4), `float`
**JS:** `typeNextChar()` — 30~70ms 랜덤 딜레이 타이핑, 3개 댓글 순환

---

### SECTION 2: 문제 공감 (STEP 1/2/3)

**디자인**: 순차 스토리텔링 + 인터랙티브 UI

**공통:**
- 카드: `bg-white rounded-2xl shadow-sm border border-gray-100`
- 상단 다크 바: `bg-gray-900` + STEP 번호 + 우측 코멘트
- 카드 사이 세로 연결선 + 전환 텍스트
- `.pain-card` 스크롤 fade-in (IntersectionObserver)

#### STEP 1: 서이추 보내도 답 없음
- 헤드라인: "서이추 100명 보냈는데, 돌아온 건... **광고성 댓글**" (빨간 font-black)
- ✉️ 발송 완료 카드 → 점 3개 기다림 UI (animate-pulse) → 광고성 답장 노란 강조 카드
- 메시지 버블 순차 등장 (700ms 간격, IntersectionObserver)

#### STEP 2: 방문자 급락
- 헤드라인: "노가다로 올린 방문자... **하루 쉬니까 곤두박질**" (빨간 font-black)
- SVG 꺾은선 그래프 (`viewBox 400x160`)
  - 7개 데이터 포인트, 스크롤 시 순차 그려짐 (350ms 간격)
  - 초록→빨강 색상 전환, 급락 점선 화살표
  - 427→34 비포/애프터 카드
- 라벨: "소통함 / 최고점 / 쉰 날 / 급락"

#### STEP 3: 번아웃
- 네이버 블로그 에디터 UI (맥 dots + "제목을 입력하세요" text-2xl)
- 커서 깜빡이다가 1.5초 후 알림 카드 3개가 덮어씀
  - 🔔 서이추 100명 미발송 (text-base)
  - 💬 답방 43개 밀림 (text-base)
  - 😫 글 한 줄 못 쓴 채 3시간 소통만 (text-lg font-black 빨간색)
- 하단: "글자 수: 0자 / 저장 안 됨"

### 공감 전환 섹션

**디자인**: 감정 전환점 — 배경 크림 그라데이션 (`from-[#F5F5F5] via-[#faf9f6] to-[#f5f2ec]`)

**구성:**
- "여러분은 충분히 열심히 했습니다." (3xl~4xl font-black)
- "지속이 불가능한 '방식'이었습니다." (naver 강조)

**BEFORE 카드 — 캐릭터 여정 타임라인:**
- 4단계 가로: ⌨️ 소통 노동 → 😵‍💫 에너지 고갈 → 😮‍💨 번아웃 → 🚪 포기
- 원형 아이콘 (`border-[3px]`) + 빨강→회색 그라데이션 연결선
- 마지막 "포기" opacity-60 회색 처리

**AFTER 카드:**
- ✨ + "이 모든 걸 대신해줄 방법이 있다면?" (2xl~3xl)
- `bg-gradient-to-r from-green-50 to-white`, `border-2 border-naver/20`

### SECTION 3: 해결책 제시

**디자인**: 연초록 배경 (`bg-naverLight`)

**헤드라인:** ✨ "서이추부터 정성 댓글까지, 블로그 소통의 **'완전 자동화'**가 시작됩니다."
**서브카피:** "손으로 직접 하던 그 복잡한 과정들, 더 이상 애쓰지 마세요."

**4단계 순환 그리드 (2x2):**
- 중앙 순환 아이콘 (AI 초록 원) + 점선 원형 보더
- 각 카드: `rounded-3xl shadow-md p-6~7 hover:shadow-lg`
  - STEP 1 타겟 분석: 적합 ✓ / 제외 ✕ 미니 리스트 (프로필 + 활동정보)
  - STEP 2 서이추 신청: 발송 완료 ✓ 리스트 + "외 48명 자동 발송 중.."
  - STEP 3 정성 댓글: AI 댓글 말풍선 (자연스러운 문체 + `..`)
  - STEP 4 이웃 관리: 블랙리스트(빨강) + 자동 취소(노랑) pill 뱃지
- 제목: `text-2xl font-black` + 구분선 + 미니 UI

**마무리 문구:** "버튼 한 번이면 AI가 사람처럼 자연스럽게 소통을 마치고 **100명을 채우면 조용히 퇴근합니다.**"

**CTA 버튼:** 흰 배경 + 초록→민트 그라데이션 텍스트, hover시 노란색 전환
```css
.cta-trial:hover { background:#FACC15; }
.cta-trial:hover .cta-trial-text { -webkit-text-fill-color:#1A1A1A; }
```

**6단계 더보기 (details/summary):**
- 다크 카드 (`bg-gray-900`) + AI 초록 아이콘 + "(클릭)" 뱃지
- 클릭 시 다크 배경(`#0A0A0A`) 6단계 상세 펼쳐짐
- STEP 01~06 좌우 교차 레이아웃 + 각 STEP별 시각 UI 목업
  - STEP 번호: `text-naver` (초록)
  - 설명 텍스트: `text-gray-400` 기본, 핵심 키워드만 `text-yellow-300` (노란색)
    - 01: "댓글창을 추적해 '진짜 활동 유저'"
    - 02: "서이추 차단 계정은 자동으로 걸러냅니다."
    - 03: "'알짜배기 계정'"
    - 04: "최신 포스팅을 실제로 읽고"
    - 05: "내 글을 진짜 읽었네?"
    - 06: "안전하게 퇴근합니다."
  - 시각 UI: 댓글창 하이라이트 / 필터링 ✕ / 서이추 폼 / 포스팅 키워드 / 댓글 타이핑 / 프로그레스 바

## 4. Supabase 연동

### 설정값 (교체 필요)
```js
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
```

### 플랜 매핑
```js
const PLAN_MAP = {
    free:     { plan: 'free_trial',   amount: 0 },
    '1month': { plan: 'monthly',      amount: 39000 },
    full:     { plan: 'full_package', amount: 59000 },
};
```

### orders 테이블
```sql
CREATE TABLE orders (
  id BIGSERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  plan TEXT NOT NULL,
  amount INTEGER DEFAULT 0,
  status TEXT DEFAULT '입금대기',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
