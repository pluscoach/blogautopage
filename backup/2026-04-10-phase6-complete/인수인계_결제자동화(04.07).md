추천: **별도 프로젝트 생성** (판매 주문 관리와 라이선스 관리 분리)
- 기존 프로젝트: 프로그램 라이선스 인증 전용 (licenses 테이블)

### 3.2 orders 테이블 스키마 (최종 확정본)

```sql
CREATE TABLE orders (
  id BIGSERIAL PRIMARY KEY,
  name TEXT NOT NULL,                    -- 입금자명 (매칭용)
  email TEXT NOT NULL,                    -- 프로그램 수령용
  plan TEXT NOT NULL,                     -- 'free_trial' | 'monthly' | 'full_package'
  amount INTEGER DEFAULT 0,               -- 0 | 39000 | 59000
  status TEXT DEFAULT '결제대기',         -- '결제대기' | '결제완료' | '발송완료' | '거절'
  ip TEXT,                                -- 무료체험 중복 차단용
  user_agent TEXT,                        -- 브라우저 정보 (선택)
  payment_link TEXT,                      -- 토스 결제 링크 (유료 플랜만)
  order_code TEXT UNIQUE,                 -- 입금자명 매칭용 고유 코드 (예: "홍길동-A3X7")
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_orders_email ON orders(email);
CREATE INDEX idx_orders_order_code ON orders(order_code);
```

### 3.4 무료 체험 중복 차단 RPC 함수

anon key로 직접 INSERT하면 중복 체크가 불가능하므로, Supabase RPC 함수로 구현.

```sql
CREATE OR REPLACE FUNCTION create_order(
  p_name TEXT,
  p_email TEXT,
  p_plan TEXT,
  p_amount INTEGER,
  p_ip TEXT,
  p_user_agent TEXT
) RETURNS json AS $$
DECLARE
  v_existing_count INTEGER;
  v_new_order_id BIGINT;
  v_order_code TEXT;
BEGIN
  -- 무료 체험 중복 체크
  IF p_plan = 'free_trial' THEN
    SELECT COUNT(*) INTO v_existing_count
    FROM orders
    WHERE plan = 'free_trial'
      AND (email = p_email OR ip = p_ip);

    IF v_existing_count > 0 THEN
      RETURN json_build_object(
        'success', false,
        'error', '이미 무료 체험을 신청하셨습니다.'
      );
    END IF;
  END IF;

  -- 고유 코드 생성 (이름 + 4자리 랜덤)
  v_order_code := p_name || '-' || upper(substring(md5(random()::text), 1, 4));

  -- 주문 생성
  INSERT INTO orders (name, email, plan, amount, ip, user_agent, order_code)
  VALUES (p_name, p_email, p_plan, p_amount, p_ip, p_user_agent, v_order_code)
  RETURNING id INTO v_new_order_id;

  RETURN json_build_object(
    'success', true,
    'order_id', v_new_order_id,
    'order_code', v_order_code
  );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- anon 사용자에게 함수 실행 권한 부여
GRANT EXECUTE ON FUNCTION create_order TO anon;
```

### 4.2 IP 수집 방법

**방법 A: 외부 API (간단)**
```javascript
async function getUserIP() {
  try {
    const res = await fetch('https://api.ipify.org?format=json');
    const data = await res.json();
    return data.ip;
  } catch {
    return null;
  }
}
```

**방법 B: Supabase Edge Function (권장)**
- Edge Function 내부에서 `request.headers.get('x-forwarded-for')` 로 IP 추출
- 외부 의존성 없음

### 4.3 폼 제출 로직 (슈도코드)

```javascript
async function submitForm(event) {
  event.preventDefault();

  const name = document.getElementById('form-name').value.trim();
  const email = document.getElementById('form-email').value.trim();
  const plan = document.querySelector('input[name="plan"]:checked').value;

  // 유효성 검증
  if (!name || !email) {
    showToast('모든 항목을 입력해주세요', 'error');
    return;
  }

  // IP 수집
  const ip = await getUserIP();

  // Supabase RPC 호출
  const { data, error } = await supabase.rpc('create_order', {
    p_name: name,
    p_email: email,
    p_plan: PLAN_MAP[plan].plan,
    p_amount: PLAN_MAP[plan].amount,
    p_ip: ip,
    p_user_agent: navigator.userAgent
  });

  if (error || !data.success) {
    showToast(data?.error || '오류가 발생했습니다', 'error');
    return;
  }

  // 플랜별 분기
  if (plan === 'free') {
    // 무료 체험: 완료 메시지
    showToast('무료 체험 신청이 완료되었습니다. 확인 후 이메일로 발송됩니다.', 'success');
  } else {
    // 유료 플랜: 토스 결제 링크로 리다이렉트
    const tossLinks = {
      '1month': 'https://buy.tosspayments.com/products/XXXXX',
      'full':   'https://buy.tosspayments.com/products/YYYYY'
    };
    // 입금자명 안내 모달 띄우고 → 결제 링크 이동
    showPaymentModal(data.order_code, tossLinks[plan]);
  }
}
```

- 상품명: "BlogAutoFriends 1개월 플랜"
- 가격: 39,000원
- 링크 복사

### 5.2 입금자명 매칭 이슈 해결

**문제:** 토스 결제 링크는 주문 ID를 직접 매칭해주지 않음. 판매자가 수동으로 결제자 이름과 orders 테이블의 name을 매칭해야 함.

**해결:** 주문 시 `order_code` (예: "홍길동-A3X7")를 생성하고, 구매자에게 **"토스 결제 시 이 코드를 입금자명으로 입력해주세요"** 라고 안내.

### 6.2 Supabase Edge Function 구조

주문 INSERT 시 자동으로 텔레그램 알림이 가도록 **Database Trigger + Edge Function** 조합 사용.

**Edge Function: `send-telegram-notification`**

```typescript
// supabase/functions/send-telegram-notification/index.ts
import { serve } from "https://deno.land/std/http/server.ts"

const TELEGRAM_BOT_TOKEN = Deno.env.get('TELEGRAM_BOT_TOKEN')
const TELEGRAM_CHAT_ID = Deno.env.get('TELEGRAM_CHAT_ID')

serve(async (req) => {
  const { order } = await req.json()

  const planNames = {
    'free_trial': '무료 체험',
    'monthly': '1개월 플랜 (39,000원)',
    'full_package': '풀 패키지 (59,000원)'
  }

  const message = `
🔔 *새 주문이 들어왔습니다*

👤 이름: ${order.name}
📧 이메일: ${order.email}
📦 플랜: ${planNames[order.plan]}
🎫 주문 코드: \`${order.order_code}\`
🌐 IP: ${order.ip || 'unknown'}
⏰ 시간: ${new Date(order.created_at).toLocaleString('ko-KR')}
  `.trim()

  const response = await fetch(
    `https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage`,
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: TELEGRAM_CHAT_ID,
        text: message,
        parse_mode: 'Markdown',
        reply_markup: {
          inline_keyboard: [[
            { text: '✅ 승인', callback_data: `approve_${order.id}` },
            { text: '❌ 거절', callback_data: `reject_${order.id}` }
          ]]
        }
      })
    }
  )

  return new Response(JSON.stringify({ success: true }))
})
```

### 6.3 Database Trigger

```sql
-- orders INSERT 시 자동으로 Edge Function 호출
CREATE OR REPLACE FUNCTION notify_new_order()
RETURNS TRIGGER AS $$
BEGIN
  PERFORM net.http_post(
    url := 'https://YOUR_PROJECT.supabase.co/functions/v1/send-telegram-notification',
    headers := jsonb_build_object(
      'Content-Type', 'application/json',
      'Authorization', 'Bearer YOUR_SERVICE_ROLE_KEY'
    ),
    body := jsonb_build_object('order', row_to_json(NEW))
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_notify_new_order
  AFTER INSERT ON orders
  FOR EACH ROW
  EXECUTE FUNCTION notify_new_order();
```

## 7. 5단계 — 승인 → 라이선스 자동 생성 + 이메일 발송

### 7.1 Webhook 서버 (Edge Function)

텔레그램의 "승인/거절" 버튼 클릭을 받아서 처리하는 Webhook.

**Edge Function: `telegram-webhook`**

주요 동작:
1. 텔레그램 callback_query 수신
2. `approve_<order_id>` 또는 `reject_<order_id>` 파싱
3. 승인 시:
   - orders.status → '결제완료'
   - licenses 테이블에 라이선스 자동 생성
   - 이메일 발송 (Resend 또는 SendGrid)
   - orders.status → '발송완료'
4. 거절 시:
   - orders.status → '거절'
   - 이메일 발송 (거절 사유 포함)

### 7.2 라이선스 자동 생성 로직

```typescript
async function generateLicense(order: any) {
  const key = `BAF-${order.order_code}-${generateRandomString(6)}`;

  const expiresAt = new Date();
  if (order.plan === 'free_trial') {
    expiresAt.setDate(expiresAt.getDate() + 1);  // 1일
  } else if (order.plan === 'monthly') {
    expiresAt.setMonth(expiresAt.getMonth() + 1); // 1개월
  } else if (order.plan === 'full_package') {
    expiresAt.setMonth(expiresAt.getMonth() + 2); // 2개월
  }

  const { data, error } = await supabase
    .from('licenses')
    .insert({
      key: key,
      buyer_name: order.name,
      is_active: true,
      expires_at: expiresAt.toISOString()
    })
    .select()
    .single();

  return data;
}
```

## 12. 주의사항 & 금지사항

### ⚠️ 보안
- **anon key는 공개되어도 되는 키**이지만, 반드시 RLS 정책으로 보호할 것
- **service_role key는 절대 프론트엔드에 노출 금지** (Edge Function 내부에서만 사용)
- 텔레그램 봇 토큰, 이메일 API 키는 환경변수로만 관리

### ⚠️ 데이터
- orders 테이블 수동 수정 시 신중히 (실수로 status 바꾸면 이메일 재발송 안 됨)
- 테스트 주문은 `status = 'test'` 같은 별도 값으로 표시 권장

### ⚠️ 토스페이먼츠
- 초기에는 상품 링크 방식 사용 (간단함)
- API 연동은 월 주문 50건 이상 안정화된 후 고려
- 환불은 토스페이먼츠 대시보드에서 수동 처리

### ⚠️ 금지사항
- 무료 체험 조건 완화 (악용 사례 증가)
- service_role key 프론트엔드 노출
- 라이선스 수동 생성 없이 테스트 발송 (라이선스 인증 실패)
