## 2. 현재까지 진행 상황

### 2.1. 사이트 개편 (v9 → 현재)

직원 고용 프레임으로 사이트 카피 개편 완료.

**히어로 카피**:
```
블로그 대행, 비싼 돈 주고 맡기지 마세요.
크몽 몇 백만 원짜리, 사람에게 맡기면 월 120~150만 원.
이웃 관리부터 글 작성까지, 시작도 전에 비용 스트레스.

오늘부턴, 일 잘하는 마케팅 팀장 한 명을 하루 1달러에 고용하세요.
그 직원이 할 줄 아는 일, 전부 담았습니다.
```

**섹션 흐름**:
```
히어로 → 능력 시연 (이웃 관리 STEP 1~4) → 글쓰기 (에디터 다크 섹션) →
가격 → 결제 폼 → FAQ → 푸터
```

**삭제됨**:
- PAIN POINTS 3장 (이웃신청 답없음 / 방문자 급락 / 번아웃) — 새 타겟 안 맞음
- 공감 전환 ("여러분은 충분히 열심히 했습니다") — 톤 안 맞음
- X2배 이벤트 박스 — 저가 전략용, 고가 전략으로 전환

**보존됨**:
- editor_event_slots DB 테이블 + RPC 2개 — 향후 "매달 10명 제한"에 재활용 가능

### 2.2. Meta 광고 라이브

**캠페인 구조**:
```
캠페인: 블로그 관리 판매 (활동 중)
  └─ 광고 세트: 블로그 판매 전환
      ├─ 광고 1: 블로그 광고 전환 (상위 1% 비밀 무기)  ← 1차 광고
      └─ 광고 2: 블로그 120만원 아끼는 법             ← 신규 추가
```

**광고 예산**:
- 캠페인 일일 예산 $25 → $50 증액 예정 (광고 2개로 늘어났으므로)
- Meta가 두 광고에 자동 분배

**1차 광고 (상위 1%) — 2일 기준 데이터**:
| 지표 | 값 | 평가 |
|---|---|---|
| 지출 | $45.03 | — |
| 도달 | 1,100명 | — |
| 노출 | 1,466회 | — |
| CTR(링크) | **5.05%** | 🟢 매우 우수 (한국 평균 0.9~1.5%의 3배) |
| CPC(링크) | $0.61 | 🟢 우수 |
| 도착률 | 88% | 🟢 정상 |
| 결제 | 0건 | 🔴 막힘 |

**2차 광고 (120만원 아끼는 법) — 1일 기준 데이터**:
| 지표 | 값 | 평가 |
|---|---|---|
| 지출 | $20.39 | — |
| 도달 | 353명 | — |
| 노출 | 405회 | — |
| CTR(링크) | **3.70%** | 🟡 1차보다 약함 |
| CPC(링크) | $1.36 | 🔴 1차의 2배 |
| 도착률 | 67% | 🟡 1차보다 21%p 낮음 |
| 게시물 저장 | 3건 | 🟢 관심 강도 좋음 |
| 게시물 참여 | 18건 | 🟢 좋음 |
| 결제 | 0건 | 🔴 막힘 |

**광고 카피 진단 결론**:
- 1차 카피 (상위 1%): 검정 배경 + "절대 손으로/AI로 하는 것" 통념 깨기 후킹 — CTR 5%로 검증됨
- 2차 카피 (120만원): "월 120만 원 vs 하루 2,000원" 가격 앵커 — CTR 3.7%로 1차 대비 약함
- 사장님 직감 ("월 120만원이 비현실적") → 데이터로 검증됨 (도착률 88% → 67%, 21%p 하락)
- **결제 0건 원인은 광고가 아니라 사이트 자체 마찰** (CTR/도착률 양호한데 결제만 0)

## 3. 다음 작업 — 사이트 전체 맥락 재구성

### 3.1. 사장님이 정한 방향

> "홈페이지 전체적인 맥락을 재구성할 거야. 현재는 너무 광범위한 그리고 **글쓰기가 중점으로 소개가 됐는데 이웃관리 솔루션 자체에 대한 걸 더 집중**시키고 **상업적 활용 / 주기적인 이웃관리를 명확하게 하는 타겟**을 페르소나로 진입할 거야."

**핵심 변경 방향**:
1. 글쓰기 (에디터) 중심 → **이웃 관리 솔루션 중심**으로 무게중심 이동
2. 광범위한 메시지 → **상업적·주기적 이웃 관리** 타겟 명확화
3. 페르소나 더 좁힘

### 3.2. 현재 사이트의 문제점 (재구성 사유)

**현재 사이트 구성 비율**:
- 이웃 관리 영역 (히어로 + STEP 1~4 + SAFETY): 약 30%
- **글쓰기 영역 (AI 에디터 다크 섹션): 약 50%** ← 비중 너무 큼
- 가격·FAQ·폼: 약 20%

**문제**:
- 메인 솔루션이 "이웃 관리"인데 사이트는 "글쓰기"가 더 강조됨
- "AI 블로그 에디터" 섹션이 길어서 본질 흐림
- 두 솔루션 동시 노출로 *"이게 뭐 하는 곳이지?"* 인지 부담

**증거 — 광고 데이터**:
- 광고 카피는 "이웃 관리" 톤 (상위 1% / 직원 고용 / 120만원)
- 사이트 도착하면 글쓰기 섹션이 큼 → 광고-사이트 메시지 미스매치
- 도착률 67% (광고 클릭 후 33% 이탈)

### 3.3. 새 페르소나 — 상업적·주기적 이웃 관리

v9에서 정한 페르소나는 광범위했음:
- 체험단·원고료 블로거
- 프리랜서 마케터
- 자영업자
- 대행사 (B2B)

**새 페르소나 (더 좁혀야 함)** — 사장님이 다음 채팅에서 명확히 할 부분:
- "상업적 활용" → 블로그를 돈 버는 도구로 쓰는 사람
- "주기적인 이웃 관리" → 매일·매주 정기적으로 이웃 관리해야 하는 사람

**예상 후보** (다음 채팅에서 확정 필요):
1. 체험단 블로거 (매일 글 + 이웃 관리)
2. 자영업자 (블로그 마케팅 정기 운영)
3. 대행사 운영자 (클라이언트 다수 관리)
4. 부업 블로거 (수익형 블로그 운영)

### 3.4. 작업 범위 예상

다음 채팅에서 확정해야 할 것:

**A. 페르소나 명확화**
- 1순위 / 2순위 / 보조 페르소나 결정
- 각 페르소나의 일과·페인·결제 방아쇠 구체화

**B. 사이트 구조 재배치**
- 이웃 관리 영역 강화 / 확장
- 글쓰기 영역 축소 (삭제 X, 보조 섹션으로 격하)
- "주기적 사용" 강조 섹션 신설 검토

**C. 카피 재정비**
- 히어로 — 페르소나 직격 카피
- 능력 시연 — 이웃 관리 STEP 더 디테일
- 글쓰기 영역 — 짧게, "추가 능력" 톤으로

**D. 데이터 검증**
- 사이트 변경 후 광고 데이터로 검증
- 도착률·결제율 변화 측정
