## 1. 사전 준비

### 1-1. 필요한 것 5가지

- **Meta 비즈니스 포트폴리오** (Business Portfolio)
- **광고 계정** (비즈니스 포트폴리오 소속)
- **결제 수단** (신용카드) 등록
- **Meta Pixel** 설치 (웹사이트에 추적 코드)
- **Facebook 페이지** (광고 표시용)

### 1-2. 광고 계정 상태 확인

**Meta Ads Manager** 접속: https://adsmanager.facebook.com

확인 사항:
- 우측 상단에 비즈니스 포트폴리오 이름 표시
- 광고 계정 ID 확인
- 결제 수단 등록 완료 여부
- 계정 점수 100점 (신규 계정은 제한 있을 수 있음)

## 2. Meta Pixel & CAPI 세팅

> **왜 중요한가?**
> Pixel만 쓰면 iOS 사용자 추적이 부정확합니다. CAPI(Conversions API)를 같이 쓰면 **서버에서 직접** Meta로 이벤트를 보내서 100% 정확하게 추적됩니다. 광고 ROAS(투자 대비 수익) 정확 측정에 필수입니다.

### 2-1. Meta Pixel 설치 (이미 설치된 경우 스킵)

웹사이트에 Meta Pixel 코드가 설치되어 있어야 합니다.

**확인 방법:**
1. Meta Events Manager 접속: https://business.facebook.com/events_manager
2. 데이터 세트에서 본인 Pixel 선택
3. "개요" 탭에서 최근 이벤트 수신 확인

**이벤트 상태:**
- 🟢 활성 = 정상 작동
- 🔴 비활성 = 문제 있음

### 2-2. Conversions API 액세스 토큰 발급

**경로:** Meta Events Manager → 본인 Pixel 선택 → **설정 탭**

1. "전환 API" 섹션 찾기
2. **"Dataset Quality API로 설정"** 선택 (권장 옵션)
3. **[액세스 토큰 만들기]** 버튼 클릭
4. 사용할 데이터 세트 선택
   - ⚠️ **1개만 선택** (여러 개 선택 시 보안 위험)
   - 본인 광고에 쓸 Pixel 1개만 체크
5. **[액세스 토큰 만들기]** 파란 버튼 클릭

**⚠️ 토큰 보안 수칙:**
- 발급받은 토큰은 **절대 외부에 공유하지 마세요**
- 대화창, 이메일, 메신저 등에 붙여넣기 금지
- 메모장에 임시 저장 → 서버에 등록 후 즉시 삭제
- 스크린샷에 토큰 포함되지 않도록 주의

### 2-3. CAPI 서버 통합

서버(Edge Function, Lambda, Node.js 등)에서 결제 완료 시점에 Meta로 Purchase 이벤트 전송.

**핵심 원칙:**
- **결제 확정 시점에 발사** (프론트 클릭 아님)
- 개인정보는 **SHA256 해시**로 익명화
- `event_id`로 **중복 제거** (Pixel과 CAPI 중복 방지)
- 실패해도 결제 플로우에 영향 없도록 **에러 격리**

**예시 Purchase 이벤트 페이로드:**
```json
{
  "data": [{
    "event_name": "Purchase",
    "event_time": 1234567890,
    "event_id": "주문코드",
    "action_source": "website",
    "event_source_url": "https://example.com",
    "user_data": {
      "em": ["sha256_해시된_이메일"],
      "ph": ["sha256_해시된_전화번호"]
    },
    "custom_data": {
      "value": 59000,
      "currency": "KRW"
    }
  }]
}
```

### 2-4. CAPI 검증 (테스트)

실제 운영 데이터 오염 없이 검증하는 방법.

**Step 1: 테스트 이벤트 코드 발급**
- Events Manager → 본인 Pixel → **"이벤트 테스트"** 탭
- 마케팅 채널: "웹사이트" 선택
- 자동 생성된 `test_event_code` 확인 (예: `TEST12345`)

**Step 2: 코드에 테스트 코드 임시 추가**

Purchase 이벤트 페이로드에 `test_event_code` 필드 추가:
```json
{
  "data": [...],
  "test_event_code": "TEST12345"
}
```

**Step 3: 실제 결제 플로우 한 번 돌리기**
- 본인 이메일로 테스트 주문
- 결제 확정까지 정상 플로우 완료

**Step 4: Events Manager에서 확인**
- "이벤트 테스트" 탭에 결과 표시 (30초~2분)
- 확인 항목:
  - 이벤트 이름: Purchase
  - 출처: **Server** (CAPI 정상 작동 증거)
  - 값, 통화, 이벤트 ID 정상 표시

**Step 5: 원복 (⚠️ 필수)**

테스트 완료 후 `test_event_code`를 코드에서 **반드시 제거**하세요.

**제거 안 하면:**
- 실제 광고 이벤트가 테스트로 분류됨
- Meta AI가 학습 못함 → 광고비 낭비

## 3. 광고 소재 & 카피 준비

### 3-1. 이미지 사이즈

| 사이즈 | 용도 | 필수도 |
|---|---|---|
| **1080×1080 (정사각)** | 피드 | ⭐ 필수 |
| 1080×1350 (세로) | 피드 모바일 최적화 | 🟡 권장 |
| 1200×628 (가로) | 링크 광고 | 🟢 선택 |
| 1080×1920 (세로 동영상 15초) | 릴스·스토리 | 🟢 선택 |

**💡 팁:** 처음엔 **정사각 1080×1080 1장**만 있어도 시작 가능. Meta가 자동으로 다른 배치용으로 크롭/조정합니다.

### 3-2. 카피 3종 세트

Meta Ads Manager에서 광고 만들 때 채워야 하는 필드 3개:

**1. Primary Text (기본 문구)**
- 위치: 이미지 **위**에 표시
- 길이: 3~5줄 권장
- 제일 중요함 (첫 줄이 훅)

**2. Headline (제목)**
- 위치: 이미지 **아래** 큰 글씨
- 길이: 30자 내외
- 간결하게 제품 가치 전달

**3. Description (설명)**
- 위치: 제목 아래 작은 글씨
- 길이: 30자 내외
- 보조 설명

### 3-3. 카피 작성 예시

**훅 없는 버전 (Primary Text A):**
```
[핵심 메시지 한 줄]

[고객 상황 공감 문장]
[제품으로 해결되는 이유]
```

**이벤트 훅 추가 (Primary Text B):**
```
[핵심 메시지 한 줄]

[고객 상황 공감 문장]
[제품으로 해결되는 이유]

🎁 [한정 혜택 문구]
```

### 3-4. Meta 광고 심사 주의 문구

아래 단어/표현은 **반려될 수 있음**:

- **과장 수치**: "상위 1%", "최고의", "유일한"
- **약속 표현**: "무조건", "100% 성공", "보장"
- **의학적 주장**: "치료", "완치" (건강 관련)
- **편법 뉘앙스**: "hack", "꼼수", "비밀"
- **금융 과장**: "월 1000만원", "확실한 수익"

**💡 팁:** 심사 반려가 걱정되면 **완곡한 표현**으로 바꾸세요. 그래도 일단 제출해보고 반려되면 수정하는 게 빠름.

## 4. 캠페인 생성 (3단계 구조)

Meta 광고는 항상 **3단 구조**:

```
📋 캠페인 (목표 설정)
  └─ 📦 광고 세트 (타겟 + 예산 + 기간)
      └─ 🎨 광고 (이미지 + 카피)
```

### 4-1. 캠페인 만들기 (1단)

**Meta Ads Manager → [+ 만들기] 클릭**

**캠페인 목표 선택:**
- **"판매"** ⭐ 선택 (Purchase 이벤트 최적화)
- 이유: CAPI로 연결한 Purchase 이벤트 기반으로 Meta AI가 실제 결제 가능성 높은 사람을 찾음

**설정:**

| 항목 | 값 | 비고 |
|---|---|---|
| 캠페인 이름 | 본인 규칙 (예: `사이트_2026-04-첫광고`) | 관리용 |
| 구매 유형 | 경매 | 기본값 |
| 캠페인 목표 | 판매 | 선택됨 |
| 어드밴티지+ 판매 캠페인 | **켜기** | AI 최적화 |
| 예산 전략 | 캠페인 예산 | 기본값 |
| 일일 예산 | $20~25 (약 3만원) | 처음 테스트용 |
| A/B 테스트 | **끄기** | 다이나믹 크리에이티브로 대체 |
| 특별 광고 카테고리 | 비워둠 | 금융/정치/고용/주택 아니면 해당 없음 |

**💡 예산 팁:**
- 통화가 USD로 고정되어 있으면 $22 = 약 3만원
- 환율 고려해서 원하는 한화 금액의 달러 환산값 입력

**[다음] 클릭**

### 4-2. 광고 세트 만들기 (2단)

**전환 설정:**

| 항목 | 값 |
|---|---|
| 광고 세트 이름 | 본인 규칙 |
| 전환 위치 | **웹사이트** |
| 성과 목표 | **전환수 극대화** |
| 데이터 세트 | 본인 Pixel 선택 |
| 전환 이벤트 | **구매 (Purchase)** ⭐ |
| 다이내믹 크리에이티브 | **켜기** ⭐ |

**💡 다이내믹 크리에이티브가 뭐?**
이미지·문구 여러 개 올리면 Meta AI가 조합을 자동 테스트해서 반응 좋은 조합을 찾아줍니다. 예산 작을 때 A/B 테스트 대체용으로 최적.

**예산 및 일정:**
- 시작 날짜: 오늘 또는 내일
- 종료 날짜: **설정 안 함** (수동 중지까지 계속)

**타겟 설정:**

| 항목 | 값 | 비고 |
|---|---|---|
| 위치 | 본인 타겟 국가 (예: 대한민국) | ⭐ 확인 필수 |
| 연령 | 18~65+ | 어드밴티지+ 자동 최적화 |
| 성별 | 모든 성별 | 어드밴티지+ 자동 최적화 |
| 상세 타게팅 | 비워둠 | AI에 맡김 |

**⚠️ 예상 타겟 규모 확인:**
- 우측 "예상 타겟 규모"에 숫자 표시됨
- 예: `7,200,000 - 8,400,000`
- 이 숫자는 **설정한 국가 안의 Meta 사용자 수**
- 해외 포함되는 게 아니라 국가 안에서만 해당하는 Meta 사용자 수
- 너무 좁으면 광고 효율 떨어짐 (100만 이상 권장)

**노출 위치:**
- **어드밴티지+ 설정** (자동)
- Facebook/Instagram/릴스/스토리 전 위치에 AI가 자동 분배

**[다음] 클릭**

### 4-3. 광고 만들기 (3단) — 가장 중요

**대표 계정:**

| 항목 | 값 | 주의사항 |
|---|---|---|
| Facebook 페이지 | 본인 페이지 선택 | 필수 |
| Instagram 프로필 | 본인 Instagram 또는 "Facebook 페이지 사용" | ⚠️ [트러블슈팅](#5-자주-발생하는-문제--해결) 참조 |

**광고 설정:**

| 항목 | 값 |
|---|---|
| 크리에이티브 소스 | 수동 업로드 |
| 형식 | 다이내믹 (광고 세트에서 켰으면 자동) |
| 여러 광고주의 광고 | 체크 (기본값) |

**광고 크리에이티브 (핵심):**

1. **미디어 추가** → 이미지/영상 업로드
2. **기본 문구** → 카피 A, B 둘 다 입력 (다이나믹 크리에이티브 활용)
3. **제목** → Headline 입력
4. **설명** → Description 입력

**랜딩 페이지:**
- **웹사이트 URL**: `https://본인도메인.com`

**행동 유도 (CTA 버튼):**
- **"지금 신청하기"** ⭐ (전환 목적에 최적)
- 또는: 지금 구매하기 / 더 알아보기 / 자세히 보기

**브라우저 추가 기능:**
- **"없음"** (기본값)

**어드밴티지+ 크리에이티브 최적화:**
- **켜기** (선택)
- Meta AI가 이미지 밝기/채도 자동 조정으로 CTR 개선

**추적 섹션 (필수 확인):**
- ✅ 웹사이트 이벤트: 본인 Pixel 체크됨 확인
- Pixel ID 정상 표시 확인

**우측 미리보기 확인:**
- Facebook 피드, Instagram 피드, 스토리, 릴스 모든 배치 미리보기
- 이상하면 수정 후 다시 저장

### 4-4. 게시

**[동의하고 게시]** 파란 버튼 클릭 → 광고 검토 시작

## 5. 자주 발생하는 문제 & 해결

### 5-1. 🔴 Instagram 프로필 "액세스 권한 필요" 오류

**증상:**
광고 만들기 화면에서 Instagram 프로필 선택 시 빨간 오류:
> "[계정명]의 광고를 게재할 액세스 권한이 필요합니다"

**원인 3가지:**

**원인 A: Instagram이 "UnclaimedBusinessUser" 상태**
- 비즈니스 포트폴리오에 임시 등록만 되고 정식 인증 안 됨
- 가장 흔한 원인

**원인 B: 사용자 권한이 "부분적 액세스"**
- 비즈니스 포트폴리오에는 연결됐지만 관리 권한 없음

**원인 C: 광고 계정과 Instagram 계정이 직접 연결 안 됨**
- Meta는 같은 포트폴리오라도 자산 간 명시적 연결 필요

---

**해결 단계 (순서대로):**

**Step 1: Meta Business Suite 접속**

https://business.facebook.com → 본인 비즈니스 포트폴리오 선택 → **설정 (⚙️)**

**Step 2: 사용자 → 사람 확인**

좌측 사이드바: **사용자 → 사람**

확인할 것:
- "UnclaimedBusinessUser FromPool" 같은 이상한 이름의 사용자
- 본인 계정 ("YOU" 표시)

**Step 3: Instagram 계정 메뉴 확인**

좌측 사이드바: **계정 → Instagram 계정**

본인 Instagram 클릭 → 다음 확인:
- 할당된 사용자 목록
- 각 사용자 권한 (전체 관리 / 부분 액세스)

**Step 4: 본인 계정에 "전체 관리 권한" 부여**

본인 이름 옆 **[관리]** 클릭 → 권한 "전체 관리 권한"으로 변경

**Step 5: 광고 계정과 Instagram 연결**

Instagram 계정 상세 페이지에서:
- **[자산 연결]** 버튼 클릭
- "광고 계정" 선택 → 본인 광고 계정 체크
- **[연결]** 또는 **[저장]**

**Step 6: 광고 화면으로 복귀**

Meta Ads Manager로 돌아가서 Instagram 프로필 드롭다운 다시 선택 → 빨간 오류 사라짐 확인

**예상 소요 시간:** 5~15분

---

**🔀 대안 (빠른 우회):**

Instagram 연결이 복잡하거나 시간이 없으면:

- Instagram 프로필 드롭다운에서 **"Facebook 페이지 사용"** 선택
- 광고는 Facebook + Instagram 모두 노출됨
- Instagram에서는 **Facebook 페이지 이름으로 표시**됨
- 나중에 여유 있을 때 Instagram 정식 연결 가능

**💡 단, 블로그·마케팅·B2C 제품은 Instagram 노출 비중이 높으므로 정식 연결 권장.**

### 5-2. ⚠️ "선택한 전환에서 수신된 활동이 없음" 경고

**증상:**
광고 세트 설정 화면에서 노란 경고:
> "선택한 전환에서 수신된 활동이 없거나 14일 이상 활동이 수신되지 않았으므로 광고가 최적화되지 않을 수 있습니다"

**해석:**
- Purchase 이벤트가 아직 Meta에 충분히 쌓이지 않음
- 보통 50건 이상 있어야 AI가 최적 학습 가능

**해결:**
- **무시하고 계속 진행** (정상적인 초기 상태)
- 광고 돌리면서 Purchase 이벤트가 쌓임 → 자동으로 개선

---

### 5-3. 🟡 캠페인 점수 93점 (100점 아님)

**증상:**
우측 상단 캠페인 점수가 100점이 아닌 93점 등.

**원인:**
Meta가 추천하는 최적화 설정(예: 어드밴티지+ 크리에이티브 개선)을 적용하지 않음.

**해결 판단:**

**"3% 결과당 비용 절감 가능" 추천이 뜨면:**
- 예산은 그대로 (일일 금액 변하지 않음)
- 같은 예산으로 구매 3% 더 나올 수 있음
- 사실상 무료 이득

**그래서:**
- **[지금 적용하기]** 클릭 → 100점 복구
- 또는 **[광고에서 보기]**로 변경 내용 미리 확인 후 결정
- 무시해도 상관없음 (93점도 충분히 좋음)

---

### 5-4. 🚫 광고 반려 (Ad Rejected)

**증상:**
광고 게시 후 이메일/알림으로 "광고가 승인되지 않았습니다" 안내.

**자주 반려되는 이유:**

| 유형 | 예시 |
|---|---|
| 과장 문구 | "상위 1%", "최고의", "100% 성공" |
| 편법 뉘앙스 | "hack", "꼼수", "비밀" |
| 의학 주장 | "치료", "완치", "효능 보장" |
| 금융 과장 | "월 1000만원 보장" |
| 개인 공격 | 특정인 비방 |
| 정책 위반 | 성인용품, 도박 등 |

**해결:**
1. 반려 이메일에 **구체적 사유** 표시됨 (어느 부분이 문제인지)
2. Meta Ads Manager → 반려된 광고 → **[수정]**
3. 문제 부분 수정 (이미지 또는 카피)
4. **[재제출]** → 다시 심사 (보통 1~6시간)

**💡 팁:**
- 반려돼도 **계정에 큰 흠 안 남음** (몇 번은 괜찮음)
- 반복 반려 시 계정 제한 가능 → 조심
- 애매하면 **완곡한 표현**으로 수정

## 6. 게시 후 모니터링

### 6-1. 게시 직후 ~ 24시간

**Meta 검토 단계:**
- 광고 상태: **"검토 중"**
- 자동 검토 (AI) + 필요시 수동 검토
- 보통 1~6시간, 길면 24시간

**검토 결과:**

| 상태 | 의미 | 사장님 행동 |
|---|---|---|
| 🟢 활성 | 승인 → 노출 시작 | 모니터링 시작 |
| 🔴 거부됨 | 반려 | [5-4](#5-4--광고-반려-ad-rejected) 참조 |
| ⏸ 검토 중 | 심사 대기 | 기다림 |

### 6-2. 첫 3일 (학습 기간)

Meta AI가 **최적 타겟 탐색** 중.

**🚫 이 기간에 절대 하지 말 것:**
- 광고 끄고 켜기 (학습 리셋)
- 예산 자주 변경 (학습 리셋)
- 타겟 수정 (학습 리셋)
- 이미지/카피 수정 (학습 리셋)

**✅ 이 기간에 봐도 되는 지표:**
- 노출 (Reach, Impressions)
- 클릭률 (CTR) — 1% 이상이면 정상
- 도달 인구 분포

**🚫 아직 판단하지 말 것:**
- CPA (구매당 비용) — 부정확
- ROAS — 데이터 부족

### 6-3. 4~7일 (안정기)

학습 완료 → 본격 성과 판단 가능.

**주요 지표 해석:**

| 지표 | 좋은 수치 | 해석 |
|---|---|---|
| CTR | 1% 이상 | 카피/이미지가 매력적 |
| CPC | 500원 이하 | 클릭당 비용 효율적 |
| CPM | 5,000~15,000원 | 1000회 노출당 비용 |
| CPA | 상품가의 10~30% | 구매당 비용 |
| ROAS | 2.0 이상 | 광고비 대비 매출 |

**💡 ROAS 계산:**
```
ROAS = 매출 ÷ 광고비

예: 광고비 30만원 쓰고 매출 60만원 → ROAS 2.0
```

### 6-4. 1주일 후 — 최적화

**A/B 카피 분석:**
- 다이나믹 크리에이티브 보고서에서 어느 조합이 반응 좋은지 확인
- 반응 좋은 조합에 예산 집중

**타겟 분석:**
- 어느 연령대/성별이 구매 많이 했는지
- 지역별 성과

**다음 주 계획:**
- 예산 유지/증액/감액 결정
- 카피 신규 버전 추가 테스트
- 새 이미지 추가

## 📌 꿀팁 모음

### 💰 예산 운영

**처음 1주차:**
- 일일 예산 **3만원 고정** (시드 데이터 축적)
- 일일 3만원 × 7일 = 21만원

**2주차:**
- 성과 좋으면 **1.5배 증액** (4.5만원)
- 성과 애매하면 **유지 + 카피 추가**
- 성과 나쁘면 **타겟/카피 전면 재검토**

**주의: 한 번에 2배 이상 증액 금지**
- Meta AI 학습 망쳐서 성과 급락 가능
- 20~30% 단계적 증액 권장

### 🎯 타겟 전략

**처음 광고:**
- **어드밴티지+ 자동 타겟** 사용
- Meta AI에게 맡기기
- 수동 설정은 데이터 쌓인 다음

**2~3주 뒤:**
- 광고 보고서에서 **반응 좋은 세그먼트 분석**
- 비슷한 세그먼트 **수동 타겟으로 추가**
- **Lookalike(유사 타겟)** 생성: "구매자와 비슷한 사람"

### 📊 대시보드 집착 금지

- 광고 대시보드는 **하루 1번만 확인**
- 실시간으로 보면 작은 변동에 감정 휘둘림
- 오전 or 저녁 고정 시간에 체크

### 🔔 주문 알림 준비

광고 효과 좋으면 **주문 폭증 가능**:

- 이메일 알림 스팸함 체크
- 전화 응대 준비
- 입금 확인 시스템 점검
- 배송/발송 가이드 최신화

## 🎯 정리: 광고 집행 전체 플로우

```
1. 준비
   ├─ Meta 비즈니스 포트폴리오 생성
   ├─ 광고 계정 + 결제 수단
   └─ Meta Pixel + CAPI 설치

2. 검증
   ├─ Pixel 정상 작동 확인
   └─ CAPI 테스트 이벤트 성공 확인 → 원복

3. 소재
   ├─ 이미지 1080×1080 준비
   ├─ 카피 2개 버전 (A: 훅 없이 / B: 이벤트 훅)
   └─ Headline + Description

4. 캠페인 (3단)
   ├─ 캠페인: "판매" + 일일 3만원
   ├─ 광고 세트: 구매 전환 + 다이나믹 크리에이티브
   └─ 광고: 이미지 + 카피 + CTA

5. 게시 → Meta 심사 → 승인 → 노출 시작

6. 모니터링
   ├─ 1~3일: 학습 기간 (건드리지 말기)
   ├─ 4~7일: 성과 판단
   └─ 1주 후: 최적화
```

## 💬 자주 묻는 질문

**Q1. 예산이 얼마부터 광고 돌릴 수 있나요?**

A. 일일 $5 (약 7,000원)부터 가능하지만, 학습에 부족. **최소 일일 $20 (약 3만원)** 권장.

**Q2. 광고 언제 끄는 게 좋아요?**

A. 최소 1주일은 돌려야 판단 가능. 3일치 데이터로는 판단 금지.

**Q3. A/B 테스트 따로 설정해야 하나요?**

A. 다이나믹 크리에이티브 켜면 자동 A/B. 별도 설정 불필요.

**Q4. Instagram 없이 Facebook만 해도 되나요?**

A. 가능. 하지만 Instagram 노출 비중이 높은 업종(B2C, 블로그, 패션 등)은 Instagram 꼭 연결 권장.

**Q5. 광고 반려되면 계정 문제 생기나요?**

A. 1~2번은 괜찮음. 반복 반려 시 계정 제한 가능. 반려 사유 꼼꼼히 보고 수정.

**Q6. CAPI 없이 Pixel만 써도 되나요?**

A. 가능. 하지만 iOS 사용자 추적 정확도 ~70%로 떨어짐. CAPI 있으면 95%+. 광고비 많이 쓸 계획이면 필수.

**Q7. 광고 결과를 어떻게 개선하나요?**

A. 4단계:
1. **CTR 낮으면** → 이미지/카피 수정 (1% 미만이면 문제)
2. **CPC 높으면** → 타겟 좁히기 또는 시간대 조정
3. **전환율 낮으면** → 랜딩 페이지 개선 (광고는 잘 먹히는데 결제 안 함)
4. **CPA 높으면** → 가격/혜택 재조정 또는 더 저렴한 타겟 찾기
