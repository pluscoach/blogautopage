## 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|----------|
| 프론트엔드 | HTML + Tailwind CDN + Vanilla JS | 빌드 도구 없이 GitHub Pages 즉시 배포 |
| 폰트 | Pretendard (CDN) | 한국어 최적화 |
| 백엔드 | Supabase (PostgreSQL) | 기존 라이선스 시스템과 동일 프로젝트. 무료 티어 |
| RPC | plpgsql SECURITY DEFINER | RLS 우회 + 서버사이드 비즈니스 로직 |
| Trigger | pg_net + Vault | DB INSERT → Edge Function 자동 호출 |
| Edge Function | Deno/TypeScript | Supabase 네이티브 |
| 카카오톡 알림 | "나에게 보내기" API | 심사 불필요 |
| 텔레그램 알림 | Bot API | 카카오톡 백업 + Phase 12에서 결제 승인 채널로 확장 |
| 이메일 | Resend | 무료 월 3000건, 도메인 인증 간단 |
| 배포 | GitHub Pages + 카페24 CNAME | 무료, 기존 사이트 영향 0 |

## 발생한 오류와 해결법

### 6.1 supabase link 권한 거부
`supabase logout` → `supabase login` 올바른 계정으로 재인증.

### 6.2 마이그레이션 버전 충돌
파일명을 `20260409_001_xxx.sql` → `20260409000100_xxx.sql` 고유 타임스탬프로 변경.

### 6.3 RPC 스키마 접근 거부
`set_config('search_path', 'public', true)` 추가.

### 6.4 Trigger → Edge Function 401
Vault에서 service_role_key를 Authorization 헤더로 전달.

### 6.5 카카오 토큰 401 → 자동갱신 미작동
`kakao_tokens` 테이블에서 refresh_token으로 자동 갱신 후 재시도.

### 6.6 Resend From 헤더 한글 깨짐
RFC 2047 Base64 인코딩 (`=?UTF-8?B?...?=`).

### 6.7 무료체험 이중 INSERT
`SELECT id INTO v_existing ... LIMIT 1` + `IF v_existing IS NOT NULL THEN RAISE EXCEPTION`.

### 6.8 CDN Supabase SDK 전역 변수
`@supabase/supabase-js@2` CDN은 `window.supabase` 전역 변수 노출.

### 6.9 GitHub Pages Custom Domain HTTPS
Settings → Pages → Custom domain에 `blog.pluscoach.co.kr` 입력 후 "Enforce HTTPS" 체크.

### 6.10 PayApp 최소 결제 금액
1원 테스트 → 에러 50054. 최소 1,000원.

### 6.11 on-new-order 유료 알림 중복
유료 플랜 조기 반환 추가 → payapp-webhook에서만 알림.

### 6.12 페이앱 cancel (pay_state=9) 처리
`pay_state !== 4` → 무시. 처리 안 함.

### 6.13 페이앱 모바일 redirectpay
`PayApp.setDefault('redirectpay', '1')` → 모바일에서 팝업 대신 리다이렉트.

### 6.14 orders.plan CHECK 제약에 lifetime 누락
`ALTER TABLE ... ADD CONSTRAINT` 로 lifetime 추가.

### 6.15 카카오 플로팅 버튼 레이아웃 깨짐
`position: fixed !important` + flex gap으로 재작성.

## 재사용 템플릿

### Supabase RPC 패턴
```sql
CREATE OR REPLACE FUNCTION public.my_rpc(p_param1 TEXT, p_param2 INTEGER)
RETURNS JSON LANGUAGE plpgsql SECURITY DEFINER AS $$
DECLARE v_result JSON;
BEGIN
    -- 비즈니스 로직
    INSERT INTO public.my_table (...) VALUES (...) RETURNING row_to_json(my_table.*) INTO v_result;
    RETURN v_result;
END;
$$;
GRANT EXECUTE ON FUNCTION public.my_rpc(TEXT, INTEGER) TO anon;
```

### Trigger → Edge Function 패턴
```sql
CREATE OR REPLACE FUNCTION public.notify_on_insert() RETURNS TRIGGER AS $$
DECLARE v_key TEXT;
BEGIN
    SELECT decrypted_secret INTO v_key FROM vault.decrypted_secrets WHERE name = 'service_role_key' LIMIT 1;
    PERFORM net.http_post(
        url := 'https://[PROJECT_REF].supabase.co/functions/v1/[FUNCTION_NAME]',
        headers := jsonb_build_object('Content-Type', 'application/json', 'Authorization', 'Bearer ' || v_key),
        body := jsonb_build_object('record', row_to_json(NEW)::jsonb)
    );
    RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_table_inserted AFTER INSERT ON public.[TABLE_NAME]
    FOR EACH ROW EXECUTE FUNCTION public.notify_on_insert();
```

### 프론트 → Supabase RPC 패턴
```js
const supabaseClient = supabase.createClient(APP_CONFIG.supabase.url, APP_CONFIG.supabase.anonKey);
const { data, error } = await supabaseClient.rpc('create_order', { p_name, p_email, ... });
if (error) throw new Error(error.message); // RAISE EXCEPTION 메시지가 담겨옴
```
