## 2. 데이터 흐름

### 무료체험
1. 사용자 폼 제출 → `create_order` RPC → INSERT (status='처리중')
2. Trigger → on-new-order → 3중 알림 + 라이선스 발급 + 인증키 이메일
3. 프론트 성공 토스트

### 유료 무통장 입금 (현재 활성)
1. 사용자 폼 제출 (이름/이메일/전화/플랜) → `create_order` RPC → INSERT (status='결제대기')
2. Trigger → on-new-order → 사장님 텔레그램 알림 (버튼 포함) + message_id 저장
3. 프론트 무통장 모달 표시 (계좌 정보 + 빨간 버튼)
4. 고객 빨간 버튼 → notify-deposit-confirmed → status='입금대기' + 답장 알림
5. 사장님 텔레그램 [✅ 승인] → telegram-callback → 라이선스 발급 + 이메일 발송

### 유료 페이앱 결제 (PAYMENT_MODE='payapp' 시)
1. 사용자 폼 제출 → `create_order` → INSERT (status='결제대기')
2. Trigger → on-new-order → 조기 반환
3. 프론트 PayApp 결제창 → 결제 완료 → payapp-webhook → 라이선스 + 이메일

## 4. DB 구조 (현재)

### 테이블

| 테이블 | RLS | 용도 |
|--------|-----|------|
| `licenses` | ON (기존 정책) | 프로그램 라이선스 — **절대 수정 금지** |
| `orders` | ON (정책 없음, RPC만) | 주문 관리 |
| `kakao_tokens` | ON (service_role만) | 카카오 토큰 단일 행 |

### orders 주요 컬럼

| 컬럼 | 타입 | 용도 |
|------|------|------|
| id | BIGSERIAL PK | - |
| name, email | TEXT NOT NULL | 고객 정보 |
| phone | TEXT DEFAULT '' | 전화번호 (Phase 12) |
| plan | TEXT CHECK (free_trial/monthly/full_package/lifetime) | 플랜 |
| amount | INTEGER | 금액 |
| status | TEXT DEFAULT '결제대기' | 처리중/결제대기/입금대기/결제완료/발급실패/이메일실패/데이터오류/금액불일치 |
| order_code | TEXT UNIQUE | {이름}-{랜덤4자리} |
| license_key | TEXT | 발급된 라이선스 키 |
| telegram_notice_message_id | BIGINT NULL | 텔레그램 알림 답장 연결용 (Phase 12) |
| ip, user_agent | TEXT | 접속 정보 |
| created_at, updated_at | TIMESTAMPTZ | 시각 |

### RPC 함수

| 함수 | 권한 | 파라미터 |
|------|------|---------|
| `create_order` | anon | p_name, p_email, p_plan, p_amount, p_ip, p_user_agent, **p_phone** (7개) |
| `create_license` | service_role | p_buyer_name, p_plan, p_order_code |
| `verify_and_activate` | anon | 기존 — 프로그램 라이선스 검증 |

### Trigger

| 트리거 | 이벤트 | 함수 | payload |
|--------|--------|------|---------|
| on_order_inserted | AFTER INSERT | notify_new_order() | id, name, email, **phone**, plan, amount, status, order_code, ip, user_agent, created_at |
| on_orders_updated | BEFORE UPDATE | handle_updated_at() | updated_at 자동 갱신 |
