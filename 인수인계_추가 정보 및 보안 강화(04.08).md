## 1. 전체 그림 요약

### 시스템 구성

```
[사용자 브라우저]
    ↓ 폼 제출
[GitHub Pages: 랜딩페이지]
    ↓ create_order RPC
[Supabase 프로젝트 B: 주문 관리]
    ├── orders 테이블
    ├── create_order RPC (anon 호출)
    └── Database Trigger
         ↓ 자동 호출
[Edge Function: send-telegram-notification]
    ↓ 텔레그램 Bot API
[사장님 텔레그램]
    ↓ [✅ 승인] 클릭
[Edge Function: telegram-webhook]
    ├─→ orders status 업데이트
    ├─→ Supabase 프로젝트 A의 create_license RPC 호출 (service_role)
    │   └── [Supabase 프로젝트 A: 라이선스]
    │       └── licenses 테이블 INSERT
    └─→ Resend API → 이메일 발송
    ↓
[사용자 이메일 수신]
    → 프로그램 다운로드 → 인증키 입력 → 사용 시작
```

### 판매 플로우

```
1. 광고/검색 → 랜딩페이지 진입
2. 플랜 선택 (무료체험/1개월/풀패키지)
3. 폼 제출 (이름/이메일/플랜)
4. 무료체험이면: 완료 메시지만
   유료면: order_code 안내 모달 + 토스 결제 링크
5. 토스에서 카드결제 (입금자명 = order_code)
6. 사장님 텔레그램에 알림 수신
7. 사장님이 토스 대시보드에서 입금 확인
8. 텔레그램 [✅ 승인] 클릭
9. 자동으로 라이선스 생성 + 이메일 발송
10. 사용자 프로그램 다운로드 → 인증 → 사용
```

### 사람이 손대는 구간은 단 하나

**사장님의 텔레그램 [✅ 승인] 버튼 클릭.**
- 토스 대시보드에서 입금자명 매칭 (5초)
- 텔레그램 승인 버튼 클릭 (3초)
- 나머지는 전부 자동

## 3. 프로그램 보안 패치 히스토리

### 배경 — 발견된 취약점

기존 구조에서 심각한 보안 문제 발견:

1. **anon key가 `db/auth_client.py`에 평문 하드코딩** (24개 파일에 복제)
2. **PyArmor 난독화 실패** (한글 인코딩 에러) → 평문 .py 파일 그대로 배포
3. **licenses 테이블에 RLS 정책 없음**
4. **구매자가 메모장으로 파일 열면 키 즉시 추출 가능**
5. **키 추출 시 SELECT/INSERT/UPDATE/DELETE 전부 가능** (라이선스 시스템 완전 무력화 위험)

**위험도: 10/10 — 배포 전 즉시 조치 필요 판단**

### 해결 — RPC + RLS 방식으로 전환

**핵심 아이디어:**
> anon key는 노출돼도 OK. 단, 그 키로 할 수 있는 일을 Supabase 쪽에서 제한.

**변경 사항:**
1. licenses 테이블에 RLS 활성화
2. anon은 licenses 직접 접근 완전 차단
3. 대신 RPC 함수 3개만 anon에게 허용
4. `auth_client.py`의 SELECT/UPDATE를 RPC 호출로 교체

### 만든 RPC 함수 3개

#### (1) verify_and_activate(p_key, p_machine_id)
인증키 검증 + 세션 활성화를 한 번에 처리.

```sql
CREATE OR REPLACE FUNCTION verify_and_activate(
  p_key TEXT,
  p_machine_id TEXT
) RETURNS json AS $$
DECLARE
  v_license RECORD;
BEGIN
  SELECT * INTO v_license FROM licenses WHERE key = p_key;

  IF NOT FOUND THEN
    RETURN json_build_object('success', false, 'error', 'invalid_key');
  END IF;

  IF v_license.is_active = false THEN
    RETURN json_build_object('success', false, 'error', 'inactive');
  END IF;

  IF v_license.expires_at IS NOT NULL AND v_license.expires_at < NOW() THEN
    RETURN json_build_object('success', false, 'error', 'expired');
  END IF;

  -- 동시접속 체크 (머신 ID가 있을 때만, 관대한 처리)
  IF p_machine_id IS NOT NULL
     AND p_machine_id != ''
     AND v_license.last_machine_id IS NOT NULL
     AND v_license.last_machine_id != p_machine_id
     AND v_license.last_active_at IS NOT NULL
     AND v_license.last_active_at > NOW() - INTERVAL '5 minutes' THEN
    RETURN json_build_object('success', false, 'error', 'another_device');
  END IF;

  -- 세션 활성화
  UPDATE licenses
  SET last_machine_id = COALESCE(NULLIF(p_machine_id, ''), last_machine_id),
      last_active_at = NOW()
  WHERE key = p_key;

  RETURN json_build_object(
    'success', true,
    'buyer_name', v_license.buyer_name,
    'expires_at', v_license.expires_at
  );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

#### (2) update_session(p_key, p_machine_id)
세션 갱신 (주기적 호출).

```sql
CREATE OR REPLACE FUNCTION update_session(
  p_key TEXT,
  p_machine_id TEXT
) RETURNS json AS $$
BEGIN
  IF p_machine_id IS NOT NULL AND p_machine_id != '' THEN
    UPDATE licenses
    SET last_active_at = NOW()
    WHERE key = p_key AND last_machine_id = p_machine_id;
  ELSE
    UPDATE licenses
    SET last_active_at = NOW()
    WHERE key = p_key;
  END IF;

  IF NOT FOUND THEN
    RETURN json_build_object('success', false, 'error', 'session_invalid');
  END IF;

  RETURN json_build_object('success', true);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

#### (3) clear_session(p_key)
세션 해제.

```sql
CREATE OR REPLACE FUNCTION clear_session(
  p_key TEXT
) RETURNS json AS $$
BEGIN
  UPDATE licenses
  SET last_machine_id = NULL,
      last_active_at = NULL
  WHERE key = p_key;

  RETURN json_build_object('success', true);
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

**create_license 설계 (참고용, 아직 구현 X):**
```sql
CREATE OR REPLACE FUNCTION create_license(
  p_buyer_name TEXT,
  p_plan TEXT,
  p_order_code TEXT
) RETURNS json AS $$
DECLARE
  v_key TEXT;
  v_expires_at TIMESTAMPTZ;
BEGIN
  -- 키 생성: BAF-{order_code}-{랜덤6자}
  v_key := 'BAF-' || p_order_code || '-' || upper(substring(md5(random()::text), 1, 6));

  -- 만료일 계산
  IF p_plan = 'free_trial' THEN
    v_expires_at := NOW() + INTERVAL '1 day';
  ELSIF p_plan = 'monthly' THEN
    v_expires_at := NOW() + INTERVAL '1 month';
  ELSIF p_plan = 'full_package' THEN
    v_expires_at := NOW() + INTERVAL '2 months';
  ELSE
    RETURN json_build_object('success', false, 'error', 'invalid_plan');
  END IF;

  INSERT INTO licenses (key, buyer_name, is_active, expires_at)
  VALUES (v_key, p_buyer_name, true, v_expires_at);

  RETURN json_build_object(
    'success', true,
    'license_key', v_key,
    'expires_at', v_expires_at
  );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- service_role만 호출 가능 (anon은 GRANT 안 함)
```

```sql
  id BIGSERIAL PRIMARY KEY,
  name TEXT NOT NULL,                    -- 입금자명 매칭용
```

**RPC 함수: create_order**
```sql
CREATE OR REPLACE FUNCTION create_order(
  p_name TEXT,
  p_email TEXT,
  p_plan TEXT,
  p_amount INTEGER,
  p_ip TEXT,
  p_user_agent TEXT
) RETURNS json AS $$
DECLARE
  v_existing_count INTEGER;
  v_new_order_id BIGINT;
  v_order_code TEXT;
BEGIN
  -- 입력 검증
  IF p_name IS NULL OR trim(p_name) = '' THEN
    RETURN json_build_object('success', false, 'error', 'name_required');
  END IF;

  IF p_email IS NULL OR trim(p_email) = '' THEN
    RETURN json_build_object('success', false, 'error', 'email_required');
  END IF;

  IF p_plan NOT IN ('free_trial', 'monthly', 'full_package') THEN
    RETURN json_build_object('success', false, 'error', 'invalid_plan');
  END IF;

  -- 무료 체험 중복 체크 (이메일 + IP)
  IF p_plan = 'free_trial' THEN
    SELECT COUNT(*) INTO v_existing_count
    FROM orders
    WHERE plan = 'free_trial'
      AND (email = p_email OR (ip = p_ip AND p_ip IS NOT NULL));

    IF v_existing_count > 0 THEN
      RETURN json_build_object(
        'success', false,
        'error', 'duplicate_free_trial',
        'message', '이미 무료 체험을 신청하셨습니다.'
      );
    END IF;
  END IF;

  -- 고유 코드 생성: 이름 + 4자리 랜덤
  v_order_code := p_name || '-' || upper(substring(md5(random()::text), 1, 4));

  -- 주문 생성
  INSERT INTO orders (name, email, plan, amount, ip, user_agent, order_code)
  VALUES (p_name, p_email, p_plan, p_amount, p_ip, p_user_agent, v_order_code)
  RETURNING id INTO v_new_order_id;

  RETURN json_build_object(
    'success', true,
    'order_id', v_new_order_id,
    'order_code', v_order_code
  );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

GRANT EXECUTE ON FUNCTION create_order TO anon;
```

### 보안 원칙
- anon key는 공개돼도 OK (RLS가 막음)
- service_role key는 Edge Function 환경변수에만 저장
- anon은 RPC 함수 호출만 가능, 테이블 직접 접근 불가
- 모든 함수는 `SECURITY DEFINER`로 RLS 우회

```javascript
const { createClient } = supabase;
const supabaseClient = createClient(
  APP_CONFIG.supabase.url,
  APP_CONFIG.supabase.anonKey
);

async function getUserIP() {
  try {
    const res = await fetch('https://api.ipify.org?format=json');
    const data = await res.json();
    return data.ip;
  } catch {
    return null;
  }
}

async function submitForm(event) {
  event.preventDefault();

  const name = document.getElementById('form-name').value.trim();
  const email = document.getElementById('form-email').value.trim();
  const plan = document.querySelector('input[name="plan"]:checked').value;

  if (!name || !email) {
    showToast('모든 항목을 입력해주세요', 'error');
    return;
  }

  // 이메일 형식 검증
  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    showToast('올바른 이메일 형식이 아닙니다', 'error');
    return;
  }

  // 제출 버튼 비활성화
  const submitBtn = document.getElementById('submit-btn');
  submitBtn.disabled = true;
  submitBtn.textContent = '처리 중...';

  try {
    const ip = await getUserIP();
    const planInfo = APP_CONFIG.plans[plan];

    const { data, error } = await supabaseClient.rpc('create_order', {
      p_name: name,
      p_email: email,
      p_plan: planInfo.plan,
      p_amount: planInfo.amount,
      p_ip: ip,
      p_user_agent: navigator.userAgent
    });

    if (error) {
      console.error(error);
      showToast('오류가 발생했습니다. 다시 시도해주세요.', 'error');
      return;
    }

    if (!data.success) {
      showToast(data.message || '처리 중 오류가 발생했습니다.', 'error');
      return;
    }

    // 성공 → 플랜별 분기
    if (plan === 'free') {
      showToast('무료 체험 신청 완료! 이메일로 안내를 보내드립니다.', 'success');
    } else {
      // Phase 4에서 모달 호출
      openPaymentModal(data.order_code, plan);
    }

  } catch (err) {
    console.error(err);
    showToast('네트워크 오류가 발생했습니다', 'error');
  } finally {
    submitBtn.disabled = false;
    submitBtn.textContent = '지금 바로 신청하고 시간 벌기';
  }
}
```

**Edge Function: telegram-webhook (핵심 로직)**
```typescript
import { serve } from "https://deno.land/std/http/server.ts"
import { createClient } from "https://esm.sh/@supabase/supabase-js@2"

const TELEGRAM_BOT_TOKEN = Deno.env.get('TELEGRAM_BOT_TOKEN')!
const PROJECT_A_URL = Deno.env.get('PROJECT_A_URL')!
const PROJECT_A_KEY = Deno.env.get('PROJECT_A_SERVICE_ROLE_KEY')!
const PROJECT_B_URL = Deno.env.get('SUPABASE_URL')!
const PROJECT_B_KEY = Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
const RESEND_API_KEY = Deno.env.get('RESEND_API_KEY')!
const RESEND_FROM_EMAIL = Deno.env.get('RESEND_FROM_EMAIL')!

const supabaseA = createClient(PROJECT_A_URL, PROJECT_A_KEY)
const supabaseB = createClient(PROJECT_B_URL, PROJECT_B_KEY)

serve(async (req) => {
  const update = await req.json()
  const callbackQuery = update.callback_query

  if (!callbackQuery) {
    return new Response('No callback', { status: 200 })
  }

  const data = callbackQuery.data  // "approve_123" 또는 "reject_123"
  const [action, orderId] = data.split('_')

  // 주문 조회 (프로젝트 B)
  const { data: order } = await supabaseB
    .from('orders')
    .select('*')
    .eq('id', orderId)
    .single()

  if (!order) {
    await answerCallback(callbackQuery.id, '주문을 찾을 수 없습니다')
    return new Response('Order not found', { status: 404 })
  }

  if (action === 'approve') {
    // 1. orders status → 결제완료
    await supabaseB
      .from('orders')
      .update({ status: '결제완료' })
      .eq('id', orderId)

    // 2. 프로젝트 A의 create_license 호출 (service_role)
    const { data: licenseData } = await supabaseA.rpc('create_license', {
      p_buyer_name: order.name,
      p_plan: order.plan,
      p_order_code: order.order_code
    })

    if (!licenseData?.success) {
      await answerCallback(callbackQuery.id, '라이선스 생성 실패')
      return new Response('License creation failed', { status: 500 })
    }

    const licenseKey = licenseData.license_key

    // 3. orders에 license_key 저장
    await supabaseB
      .from('orders')
      .update({ license_key: licenseKey })
      .eq('id', orderId)

    // 4. Resend로 이메일 발송
    await sendLicenseEmail(order, licenseKey)

    // 5. orders status → 발송완료
    await supabaseB
      .from('orders')
      .update({ status: '발송완료' })
      .eq('id', orderId)

    // 6. 텔레그램 응답
    await answerCallback(callbackQuery.id, '✅ 승인 완료')
    await editMessage(callbackQuery.message, `✅ 승인 완료: ${order.name}\n라이선스: ${licenseKey}`)

  } else if (action === 'reject') {
    await supabaseB
      .from('orders')
      .update({ status: '거절' })
      .eq('id', orderId)

    await answerCallback(callbackQuery.id, '❌ 거절')
    await editMessage(callbackQuery.message, `❌ 거절: ${order.name}`)
  }

  return new Response('OK', { status: 200 })
})

async function sendLicenseEmail(order: any, licenseKey: string) {
  const planNames: Record<string, string> = {
    'free_trial': '무료 체험 (1일)',
    'monthly': '1개월 플랜',
    'full_package': '풀 패키지 (2개월)'
  }

  const html = `
    <h2>안녕하세요, ${order.name}님</h2>
    <p>BlogAutoFriends를 구매해주셔서 감사합니다.</p>

    <h3>📦 주문 정보</h3>
    <ul>
      <li>플랜: ${planNames[order.plan]}</li>
      <li>인증키: <code>${licenseKey}</code></li>
    </ul>

    <h3>📥 프로그램 다운로드</h3>
    <p><a href="${Deno.env.get('GOOGLE_DRIVE_DOWNLOAD_URL')}">다운로드 링크</a></p>

    <h3>💡 설치 방법</h3>
    <ol>
      <li>위 링크에서 프로그램 다운로드</li>
      <li>압축 해제 후 run.bat 실행</li>
      <li>인증키 입력란에 위 인증키 붙여넣기</li>
    </ol>

    <p>문의: 카카오톡 오픈채팅</p>
  `

  await fetch('https://api.resend.com/emails', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${RESEND_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      from: RESEND_FROM_EMAIL,
      to: order.email,
      subject: '[BlogAutoFriends] 프로그램 & 인증키 안내',
      html: html
    })
  })
}

async function answerCallback(callbackId: string, text: string) {
  await fetch(`https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/answerCallbackQuery`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ callback_query_id: callbackId, text })
  })
}

async function editMessage(message: any, newText: string) {
  await fetch(`https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/editMessageText`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      chat_id: message.chat.id,
      message_id: message.message_id,
      text: newText
    })
  })
}
```

## 11. 주의사항 & 금지사항

### ⚠️ 보안

**절대 하지 말 것:**
- ❌ `service_role key`를 프론트엔드(config.js, HTML, JS)에 노출
- ❌ Resend API key 프론트엔드 노출
- ❌ 텔레그램 봇 토큰 프론트엔드 노출
- ❌ 프로그램 `auth_client.py`의 RPC 안 쓰고 테이블 직접 접근
- ❌ RLS 끄고 anon에게 테이블 전체 권한 부여

**반드시 할 것:**
- ✅ anon key는 RLS + RPC로 보호
- ✅ service_role key는 Edge Function 환경변수에만
- ✅ 모든 민감 정보는 `.gitignore`
- ✅ Supabase 대시보드 2FA 활성화

### ⚠️ 데이터

- ❌ orders 테이블 수동 수정 시 주의 (status 잘못 바꾸면 이메일 재발송 안 됨)
- ❌ 테스트 주문은 운영과 섞이지 않게 (`email: test@...` 같은 패턴)
- ❌ licenses 테이블에 수동 INSERT 시 key 중복 주의

### ⚠️ 결제

- ❌ 토스 API 연동은 초기에 하지 말 것 (월 50건 이후 고려)
- ❌ 환불은 토스페이먼츠 대시보드에서 수동 처리
- ❌ 무료 체험 조건 완화하지 말 것 (악용 급증 위험)

### ⚠️ 프로그램

- ❌ `auth_client.py`의 함수 시그니처 변경 금지 (main_ui.py 호환성)
- ❌ 운영 DB의 licenses 테이블 수정 시 RLS 정책 확인
- ❌ 이미 배포된 사용자 있으면 SUPABASE URL/KEY 변경 시 전원 마비

### ⚠️ 법적

- ❌ 개인정보처리방침 없이 이메일 수집 금지
- ❌ 환불 규정 명시 없이 "100% 환불 보장" 광고 금지
- ❌ 통신판매업 신고 주소 미변경 상태로 장기 운영 금지 (강동구청 주소 변경 필요)
